Recursos

  • Arquivos adicionais contendo informações pré-definidas a ser utilizada para organizar a interface do app
  • Incluem: bitmaps, layouts, preferências de interface, animações, texto, etc.
  • Presentes na pasta res

Os recursos suportados

Pasta Recurso suportado
animator Propriedades de animações
anim Animações de interface
color Paleta de cores a ser utilizada no aplicativo
mipmap Ícones a serem utilizados no app
drawable Arquivos de imagem (em formato tradicional ou XML)
layout Arquivos de layout
menu Definição de listas e menu
raw Outros tipos de recursos
value Valores e propriedades textuais simples, como texto, números, tamanhos, cores de componentes, etc
xml Outros arquivos XML a serem utilizados no app, ex: configurações
font Fontes textuais

layout

  • Armazena todos os arquivos de UI utilizados no aplicativo (layout, componentes, etc)
  • Pode ser recuperado via código pela variável R.layout.nome_do_arquivo

value

  • Contém a definição de valores básicos a serem utilizados no aplicativo, como string, color, e dimension
  • Convenção: cria-se um arquivo por tela/componente
  • Nomenclatura: <tipo do recurso>_<nome da tela>.xml

value: strings


            <resources>
                <string name="welcome_msg">Olá!!</string>
                <string name="action_settings">Preferências</string>
            </resource>
        

            <TextView android:text="@string/welcome_msg" 
                android:layout_width="wrap_content" 
                android:layout_height="wrap_content"
                android:textSize="18dp" />

value: colors

Pode ser definida por arquivo e/ou por aplicação. 


            <?xml version="1.0" encoding="utf-8"?>
            <resources>
                <color name="text_dark">#252525</color>
                <color name="text_light">#DEDEDE</color>
                <color name="text_warn">#FF0000</color>
            </resources>
        

            <TextView android:text="@string/welcome_msg"
                android:textColor="@color/text_warn"
                android:layout_width="wrap_content" 
                android:layout_height="wrap_content"
                android:textSize="18dp" />

value: dimension

Assim como as cores, pode ser definida por arquivo e/ou por aplicação. 


            <resources>
                <dimen name="activity_horizontal_margin">16dp</dimen>
                <dimen name="activity_vertical_margin">16dp</dimen>
                <dimen name="default_margin">8dp</dimen>
            </resources>
        

            <TextView android:text="@string/welcome_msg"
                android:textColor="@color/text_warn"
                android:layout_width="wrap_content" 
                android:layout_height="wrap_content"
                android:layout_margin="@dimen/default_margin"
                android:textSize="18dp" />

drawable

  • Pasta reservada para armazenar imagens que serão utilizadas pelo aplicativo
  • Cada arquivo representa uma imagem específica
  • As imagens podem ser representados por arquivos de imagens tradicionais (png, jpg, etc) ou XML
  • Pode ser recuperado via código pela variável R.drawable.nome_do_arquivo
  • Pode ser recuperado via XML pela variável @drawable/nome_do_arquivo

drawable


            <ImageView    
                android:layout_height="wrap_content"    
                android:layout_width="wrap_content"    
                android:src="@drawable/ic_error" />

Recursos Alternativos

Permite a declaração de diferentes arquivos de recursos de acordo com a especificação do device.

Recursos Alternativos

Configuração Exemplo Descrição
Língua e Região en, fr, pt-br, etc Recursos disponíveis de acordo com a língua adotada no dispositivo
Menor Largura w720dp, w480dp, w<N>dp Carrega o arquivo referente a menor largura válida para a tela
Modo de UI car, desk, television, etc Recursos são disponibilizados de acordo com o modo de exibição do app
Densidade de Tela ldpi, mdpi, hdpi, etc Arquivo de recursos é selecionado de acordo com a densidade de pixels da tela

Recursos alternativos

  • São configurados por diretórios com nomenclatura específica:
    res/<tipo de recurso>-<tipo de configuração>/
  • Devem estar dentro da pasta res/
  • Devem ser usados em conjunto com as configurações default (ex: values/ e values-en/)

Exemplo: Internacionalização

Internacionalização

O Android possui suporte nativo a internacionalização

Para isso, basta criar um arquivo strings.xml por idioma

O arquivo strings.xml contém a lista de mensagens a serem traduzidas

Internacionalização


            <resources>
                <string name="btn_enviar">Send</string>
                <string name="lbl_nome">Name:</string>
                <string name="chk_emails">Receive e-mails</string>
            </resources>        
        

            <resources>
                <string name="btn_enviar">Enviar</string>
                <string name="lbl_nome">Nome:</string>
                <string name="chk_emails">Receber e-mails</string>
            </resources>
